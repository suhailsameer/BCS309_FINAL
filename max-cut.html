NP-Completeness of Max-Cut

Problem Definition & Overview ▲

Max-Cut Problem: Given an undirected graph G = (V, E) and integer k, does there exist a partition of V into sets S and V∖S such that at least k edges cross the cut?

Input: Graph G = (V, E), integer k
Question: ∃ partition (S, V∖S) such that |cut(S, V∖S)| ≥ k?
cut(S, V∖S) = { (u,v) ∈ E : u ∈ S, v ∈ V∖S }

Max-Cut arises in VLSI design (minimize wire crossings), statistical physics (Ising spin glass models), and network clustering. The celebrated Goemans-Williamson SDP achieves a 0.878 approximation, but finding the exact optimum is NP-hard.

Why It Matters

Max-Cut is one of Karp's original 21 NP-complete problems. Its intractability has practical consequences — every exact algorithm requires exponential time in the worst case.

Applications

Circuit layout, statistical physics, community detection in social networks, image segmentation, and quantum computing (QAOA algorithm targets Max-Cut).

Step 1: Max-Cut ∈ NP ▲
Certificate
A proposed solution is a partition: a subset S ⊆ V. This certificate has size O(|V|), which is polynomial.
Verifier
Given certificate S, count the edges crossing: iterate over every edge (u,v) ∈ E and check if exactly one endpoint is in S. This takes O(|E|) time — polynomial in the input size.
Conclusion
Since we can verify any claimed solution in polynomial time, Max-Cut ∈ NP. ✓
Verify(G, k, S): cut_size ← |{ (u,v) ∈ E : exactly one of u,v is in S }| return cut_size ≥ k // O(|E|)
Step 2: NAE-3-SAT ≤ₚ Max-Cut ▲

NAE-3-SAT (Not-All-Equal 3-SAT): Given a 3-CNF formula, does there exist a truth assignment where in every clause, the three literals are not all equal (not all true and not all false)?

NAE-3-SAT is NP-complete (provable from 3-SAT). We reduce it to Max-Cut.

Construction
Given NAE-3-SAT formula φ with variables x₁…xₙ and clauses C₁…Cₘ, build graph G:

• For each variable xᵢ: add nodes xᵢ and x̄ᵢ, and add edge (xᵢ, x̄ᵢ).
• For each clause (a ∨ b ∨ c): add edges (a,b), (b,c), (a,c) — forming a triangle on the three literal nodes.
• Set k = m (one cut per clause triangle is required).
Forward (⇒)
If φ is NAE-satisfiable, then G has a cut of size ≥ m + n.
Partition S: place xᵢ in S iff xᵢ = true. Since each variable-edge (xᵢ, x̄ᵢ) is always cut (one true, one false), all n variable-edges cross. For each clause triangle, NAE guarantees not all literals are equal, so at least one edge in each triangle crosses. Total cut ≥ m + n.
Backward (⇐)
If G has a large enough cut, then φ is NAE-satisfiable.
Given partition (S, V∖S) achieving the cut, assign xᵢ = true iff xᵢ ∈ S. The variable-edges ensure xᵢ and x̄ᵢ are on opposite sides. For each clause, if all three literals were on the same side, the triangle would contribute 0 cut edges — contradicting the cut bound. So each clause has literals on both sides → NAE condition satisfied. ✓
Poly-time
Construction creates O(n + m) nodes and O(n + 3m) edges — polynomial in formula size. ✓
|V| = 2n + dummy nodes = O(n) |E| = n (var-edges) + 3m (clause triangles) = O(n + m) Construction time: O(n + m) ← polynomial ✓
0.5-Approximation Guarantee (CLO-3) ▲

Claim: At a local maximum of the greedy local search, the cut value ≥ |E|/2.

Key Insight
At a local optimum, no single vertex flip improves the cut. So for each vertex v, the number of edges to the opposite side ≥ edges to the same side.
Formal Proof
For vertex v in partition S, let dₛ(v) = edges to V∖S (cut edges), dᵢ(v) = edges to S (internal). At local max: dₛ(v) ≥ dᵢ(v) for all v. Summing over all v: Σ dₛ(v) ≥ Σ dᵢ(v). Each cut edge contributes 1 to each side's sum, each internal edge contributes 2. So 2·|cut| ≥ 2·|internal|, giving |cut| ≥ |internal| ≥ |E|/2.
At local max: ∀v: dₛ(v) ≥ dᵢ(v) Sum over V: 2·|cut| ≥ 2·|internal| Since |cut| + |internal| = |E|: |cut| ≥ |E|/2 ← 0.5-approximation ✓
